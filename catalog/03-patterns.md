# Patterns

The interaction patterns Apple documents under Patterns. Names are Apple's. The
descriptions are this project's own paraphrase.

Verify the latest at
https://developer.apple.com/design/human-interface-guidelines/patterns

| Pattern | What it is | When to use, key guidance |
|---|---|---|
| Charting data | Presenting data visually | Choose the chart that fits the data; label it and make it accessible |
| Collaboration and sharing | Letting people share and co-edit | Use the share sheet and system collaboration; show presence clearly |
| Drag and drop | Moving content by dragging | Give clear drop targets and feedback; support within and across apps |
| Entering data | Forms and input | Keep typing low; validate gently; use the right keyboard and pickers |
| Feedback | Telling people what happened | Confirm actions; show progress and errors clearly; use haptics sparingly |
| File management | Browsing and organizing files | Use the system document and file pickers; respect iCloud and Files |
| Going full screen | Full-screen modes | Make entering and exiting obvious; keep essential controls reachable |
| Launching | The app launch experience | Show a fast launch screen; restore state; avoid heavy work at launch |
| Live-viewing apps | Real-time shared viewing on visionOS | Keep latency low and presence clear |
| Loading | Showing content is on the way | Use skeletons or progress; never a blank stuck screen; allow retry |
| Managing accounts | Sign in, sign up, sign out | Offer Sign in with Apple; allow easy sign-out and in-app account deletion |
| Managing notifications | Asking for and sending notifications | Ask at the right moment; keep them useful and easy to turn off |
| Modality | Modal versus non-modal flows | Use modality for a focused, self-contained task; give a clear way out |
| Multitasking | Running alongside other apps on iPad | Support Split View, Slide Over, and Stage Manager where relevant |
| Offering help | In-app help and guidance | Make help easy to find; prefer inline tips (TipKit) over manuals |
| Onboarding | The first-run experience | Reach value fast; keep steps few and skippable; teach in context |
| Playing audio | Audio playback | Use the system audio session; show now-playing; respect silent mode and routes |
| Playing haptics | Tactile feedback | Use system haptics with meaning; do not overuse; honor the person's settings |
| Playing video | Video playback | Use the system player; support Picture in Picture, captions, and AirPlay |
| Printing | Printing content | Use the system print flow; offer sensible defaults and a preview |
| Ratings and reviews | Asking for App Store ratings | Ask after a good moment; use the system prompt; do not nag |
| Searching | Finding content | Offer search with suggestions and clear results; support scopes |
| Settings | App configuration | Keep settings minimal; use system patterns; ship sensible defaults |
| Undo and redo | Reversing actions | Support undo widely; make destructive actions reversible or confirmed |
| Workouts | Fitness sessions on watchOS | Use Always On display, quick controls, and reliable tracking |
