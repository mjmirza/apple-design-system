# Foundations

The cross-cutting design topics Apple documents under Foundations. Names are
Apple's. The descriptions are this project's own paraphrase.

Verify the latest at
https://developer.apple.com/design/human-interface-guidelines/foundations

| Topic | What it is | Key rule, when it matters |
|---|---|---|
| Liquid Glass | The current translucent design language across all platforms | Lead with content; pick the glass treatment that keeps content legible; respect the opacity setting |
| Materials | System background and glass finishes, including Liquid Glass and vibrancy | Use system materials so foreground content stays legible over varied backgrounds |
| Accessibility | Designing so everyone can use the app | Support Dynamic Type, VoiceOver, contrast, and Reduce Motion; test with assistive tech |
| App icons | The icon that represents the app | Use the layered Icon Composer format; keep it simple and legible at every size |
| Branding | Expressing a brand within Apple's UI | Stay restrained, let content lead, never imitate system UI to mislead |
| Color | System and custom color | Prefer semantic system colors; meet contrast; support light and dark |
| Dark Mode | The dark appearance | Provide both appearances; test legibility and assets in each |
| Icons | Iconography beyond the app icon | Prefer SF Symbols; keep meaning clear and consistent |
| Images | Photos and graphics in the UI | Provide correct resolutions; respect aspect and safe areas; add alternative text |
| Immersive experiences | Full-space spatial experiences on visionOS | Use depth and scale responsibly; keep comfort and an easy way out |
| Inclusion | Designing for a wide range of people | Avoid bias; use inclusive language and representation |
| Layout | Arranging content and controls | Respect safe areas and margins; adapt across size classes; align to a grid |
| Motion | Animation and transitions | Use motion to clarify, not distract; honor Reduce Motion |
| Privacy | Handling personal data respectfully | Request the minimum, explain why, follow platform privacy rules |
| Right to left | Mirroring the interface for RTL languages | Mirror layout and controls; keep numerals and certain glyphs correct |
| SF Symbols | Apple's system symbol library | Use symbols for icons; match symbol weight to text; respect rendering modes |
| Spatial layout | Arranging content in 3D space on visionOS | Place content at a comfortable depth and angle; keep it within the field of view |
| Typography | Type with San Francisco and New York | Use Dynamic Type and the built-in text styles; keep hierarchy clear |
| Writing | UI text and tone of voice | Be clear, brief, and consistent; write for the person, not the system |
