# Designing for each platform

The defining design traits of each Apple platform, from the "Designing for"
theme pages. Names are Apple's. The descriptions are this project's own
paraphrase.

Verify the latest at
https://developer.apple.com/design/human-interface-guidelines/platforms

| Platform | Defining traits | What to prioritize |
|---|---|---|
| Designing for iOS | Touch-first, focused, one main task per screen, mostly portrait | Clarity, content, quick tasks, one-handed reach |
| Designing for iPadOS | More screen space, multitasking, pointer and Pencil, sidebars | Use the space, support multitasking, pointer, keyboard, and Pencil |
| Designing for macOS | Windows, the menu bar, pointer precision, higher density | The full command set, keyboard, multiple windows, precise pointer |
| Designing for watchOS | Tiny screen, glanceable, Digital Crown, Always On | Brevity, complications, quick actions, Always On legibility |
| Designing for tvOS | Across-the-room, focus-based, remote input, large type | Focus clarity, large legible type, simple navigation, Top Shelf |
| Designing for visionOS | Spatial, depth, eyes and hands, windows and volumes | Comfort, depth and scale, gaze-and-pinch targets, passthrough respect |
| Designing for games | Immersive, performance-first, controller and touch | Performance, input options, accessibility, fast onboarding |
| Designing for CarPlay | In-car, driving-safe, glanceable, voice-first | Few glances, large targets, Siri, the provided templates |
